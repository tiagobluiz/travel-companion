# Component Guidelines

## Buttons
Variants:
1. Primary
2. Secondary
3. Ghost
4. Destructive

States:
1. Default
2. Hover
3. Focus
4. Disabled
5. Loading

## Inputs and Selects
Rules:
1. Label always visible.
2. Helper text optional.
3. Error text shown below control.
4. Success and error are not color-only; include icon/text cues.

## Cards
Use cards for trip summaries and itinerary containers.
1. Clear heading.
2. Metadata row.
3. Action area separated visually.

## Tabs
1. Primary tabs for app-level destinations.
2. Secondary tabs for trip sub-sections.
3. Active state must be obvious and keyboard reachable.

## Modals and Bottom Sheets
Use for:
1. Auth gates.
2. Confirmations.
3. Quick forms on mobile.

Requirements:
1. Focus trap.
2. Esc to close on desktop.
3. Close action always visible.

## Alerts and Toasts
1. Inline alerts for form and section-level issues.
2. Toasts for transient success notices.
3. Error alerts should include recovery action when possible.

## Empty States
Every empty state includes:
1. Context title.
2. One-line explanation.
3. Primary CTA.
4. Optional secondary CTA.
